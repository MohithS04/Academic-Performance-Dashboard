# Real-Time Academic Performance Dashboard 🎓

A comprehensive, real-time academic tracking dashboard built with Python, Streamlit, and SQLite. This project simulates a live data environment, enabling school administrators to monitor student enrollment, track daily attendance, and visualize academic performance metrics dynamically.

It also highlights automated data validation workflows to ensure data integrity, acting as an end-to-end simulation of a production reporting system.

## Features

*   **Real-Time Data Simulation**: A background script continuously generates synthetic student events (enrollments, attendance logs, and exam grades) and inserts them into an SQLite database to mimic live traffic.
*   **Executive Overview**: High-level KPI cards displaying Total Enrollment, Average GPA, and Overall Attendance Rate, updating automatically.
*   **Enrollment & Attendance Tracking**: Interactive line charts and pie charts tracking school growth and daily attendance distributions.
*   **Performance Analytics**: Grade distributions broken down by subject and a live feed of the top-performing students.
*   **Automated Data Validation**: A dedicated section that actively monitors the simulated data for deliberate "anomalies" injected by the simulator (e.g., negative grades or orphaned attendance records without a matching student ID), ensuring data integrity workflows are visible.

## Technology Stack

*   **Frontend / Dashboard**: [Streamlit](https://streamlit.io/)
*   **Data Visualization**: [Plotly](https://plotly.com/python/)
*   **Database**: SQLite (Local file-based database for simplicity)
*   **Mock Data Generation**: [Faker](https://faker.readthedocs.io/en/master/)

## Project Structure

```text
├── DB_FILE                 # academic_data.db (Auto-generated by the simulator)
├── app.py                  # Main Streamlit dashboard application
├── data_simulator.py       # Background script for real-time SQLite data generation
├── requirements.txt        # Python dependencies
└── run.sh                  # Shell script to boot both processes concurrently
```

## Setup & Installation

**Prerequisites:** Ensure you have Python 3.8+ installed on your system.

1.  **Clone the Repository** (or navigate to the project directory):
    ```bash
    cd "Realtime Projects/Academic performance Dashboard"
    ```

2.  **Make the script executable**:
    ```bash
    chmod +x run.sh
    ```

3.  **Run the application**:
    ```bash
    ./run.sh
    ```

    The `run.sh` script handles everything:
    - Installs the required pip dependencies from `requirements.txt`.
    - Boots the `data_simulator.py` script in the background.
    - Launches the `app.py` Streamlit server.
    - Automatically opens the dashboard in your default web browser at `http://localhost:8501`.

## How it Works

When you start the project, `data_simulator.py` will first seed the SQLite database with 100 mock students and historical performance data. It then enters an infinite loop, randomly picking an event type (new attendance, new grade, new student) and inserting it into the database every 2 to 5 seconds. 

Meanwhile, `app.py` runs on a loop (using `st.rerun()`), checking the database for new records based on the configured "Refresh Rate" slider in the sidebar. When the dashboard pulls new data, the visual charts and KPI metrics change to reflect the simulated real-world updates in the school district!
